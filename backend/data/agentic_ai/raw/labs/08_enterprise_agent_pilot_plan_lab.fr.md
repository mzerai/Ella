# Lab 8 — Construire le plan pilote d’un agent d’entreprise

## Mission

Vous devez construire un plan de pilote 60 ou 90 jours pour un agent IA d’entreprise.

Choisissez un agent :

- agent support interne ;
- agent admission ;
- agent RH ;
- agent finance ;
- agent conformité ;
- agent service client ;
- agent achats.

Votre objectif est de montrer que l’agent peut être testé avec de vrais utilisateurs sans prendre de risque excessif.

---

## Situation proposée

Une organisation dispose d’un prototype d’agent support interne.

L’agent peut :

- lire des tickets ;
- chercher des procédures ;
- préparer des brouillons ;
- créer des notes internes ;
- escalader les cas sensibles.

La direction veut le déployer à toute l’organisation.

Vous devez proposer un pilote limité et gouverné.

---

## Contraintes

Votre plan doit :

- refuser un déploiement large immédiat ;
- commencer par un périmètre limité ;
- inclure observation ou assistance avant action ;
- nommer des responsables ;
- prévoir communication, formation et support ;
- inclure des KPI qualité, sécurité, adoption, métier et exploitation ;
- prévoir critères d’arrêt et d’extension ;
- prévoir rollback ou désactivation ;
- préparer le passage vers le lab final.

---

## Ce que Ella va évaluer

Ella vérifiera si vous :

- structurez un pilote réaliste ;
- limitez le périmètre ;
- définissez les rôles ;
- préparez les utilisateurs ;
- prévoyez support et communication ;
- choisissez des KPI utiles ;
- identifiez risques et garde-fous ;
- définissez une décision finale claire.

---

## Indices

- Un pilote n’est pas une production.
- Un agent utile doit avoir un propriétaire.
- L’adoption se prépare.
- Un KPI sécurité peut bloquer l’extension.
- Le support pendant le pilote révèle les vrais problèmes.
